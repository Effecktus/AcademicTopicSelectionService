**2.X. Диаграмма вариантов использования программно-информационной системы**

Для формализации функциональных требований к разрабатываемой программно-информационной системе (ПИС) поддержки выбора темы выпускной квалификационной работы (ВКР) и научного руководителя построена диаграмма вариантов использования в нотации UML. На диаграмме показаны акторы, соответствующие ролям участников процесса, и варианты использования, реализуемые в границах прототипа. К функциям, не входящим в разрабатываемый контур (интеграция с электронно-информационной средой вуза, размещение текстов ВКР в электронно-библиотечной системе, проверка на объём заимствования и полный цикл государственной итоговой аттестации), варианты использования на данной диаграмме не относятся.

Состав акторов и их назначение приведён в таблице X.

**Таблица X** — Акторы программно-информационной системы


| Актор                                | Назначение                                                                                                                                                                                                               |
| ------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Студент                              | Выбор темы ВКР и научного руководителя, подача и сопровождение заявки, отмена заявки на допустимых этапах, просмотр статусов, переписка по заявке до окончательного решения заведующего кафедрой, работа с уведомлениями |
| Преподаватель (научный руководитель) | Создание и ведение тем ВКР, рассмотрение заявок студентов (одобрение или отклонение), переписка в привязке к заявке, просмотр статусов и уведомлений                                                                     |
| Заведующий кафедрой                  | Итоговое утверждение или отклонение заявки после рассмотрения научным руководителем, просмотр заявок и уведомлений                                                                                                       |
| Администратор                        | Регистрация и учёт пользователей, ведение нормативно-справочной информации (кафедры, роли, классификаторы), загрузка и ведение архива защищённых ВКР, экспорт данных для отчётности                                      |


Варианты использования согласованы с организационными положениями, регламентирующими выбор и утверждение темы ВКР и взаимодействие обучающегося с руководителем ВКР. В частности, цепочка «подача заявки — согласование с научным руководителем — утверждение заведующим выпускающей кафедрой» отражает требования к порядку согласования темы и закрепления руководителя, изложенные в положении о ВКР (п. 4.2.1, 4.2.4, 4.2.7, 4.2.8 документа П-8.2.3.2/8.3.4-01-2020). Операция одобрения или отклонения заявки на диаграмме представлена одним вариантом использования: она выполняется научным руководителем и заведующим кафедрой последовательно на разных этапах маршрута согласования, при этом с точки зрения системы это одно и то же действие над заявкой с разграничением по роли и текущему статусу. Связи стереотипа «include» с вариантом «Аутентификация и управление сеансом» показывают, что подача заявки и принятие решения по заявке выполняются только в авторизованном сеансе пользователя.

## Код для [https://www.plantuml.com/plantuml](https://www.plantuml.com/plantuml)

Скопируйте блок целиком (от `@startuml` до `@enduml`) в форму на сайте.

```
@startuml
skinparam packageStyle rectangle
skinparam actorStyle awesome
skinparam nodesep 70
skinparam ranksep 60
skinparam ArrowThickness 1.2
left to right direction

actor "Студент" as Stu
actor "Преподаватель\n(научный руководитель)" as Sup
actor "Заведующий\nкафедрой" as Head

rectangle "Программно-информационная система\nвыбора темы ВКР и научного руководителя" {

  package "Доступ и профиль" {
    usecase "Аутентификация\nи управление сеансом" as UC_Login
  }

  package "Каталог и заявки" {
    usecase "Просмотр каталога тем ВКР\nи научных руководителей" as UC_Browse
    usecase "Подача заявки на тему\nи научного руководителя" as UC_Submit
    usecase "Отмена заявки\n(на допустимых этапах)" as UC_Cancel
    usecase "Просмотр статусов\nи списка заявок" as UC_ListApp
    usecase "Одобрение или отклонение заявки\n(на этапе руководителя и заведующего)" as UC_Decide
  }

  package "Темы ВКР" {
    usecase "Создание и ведение\nтем ВКР" as UC_ManageTopics
  }

  package "Коммуникации и уведомления" {
    usecase "Переписка по заявке\n(чат до решения заведующего)" as UC_Chat
    usecase "Просмотр уведомлений" as UC_Inbox
  }

  package "Справочники и архив" {
    usecase "Регистрация и учёт\nпользователей" as UC_Users
    usecase "Ведение справочников\n(кафедры, роли, классификаторы)" as UC_Dict
    usecase "Загрузка и ведение\nархива защищённых ВКР" as UC_Archive
    usecase "Экспорт данных\nдля отчётности" as UC_Export
  }
}

actor "Администратор" as Adm

Stu --> UC_Login
Stu --> UC_Browse
Stu --> UC_Submit
Stu --> UC_Cancel
Stu --> UC_ListApp
Stu --> UC_Chat
Stu --> UC_Inbox

Sup --> UC_Login
Sup --> UC_ManageTopics
Sup --> UC_Decide
Sup --> UC_ListApp
Sup --> UC_Chat
Sup --> UC_Inbox

Head --> UC_Login
Head --> UC_Decide
Head --> UC_ListApp
Head --> UC_Inbox

Adm --> UC_Login
Adm --> UC_Users
Adm --> UC_Dict
Adm --> UC_Archive
Adm --> UC_Export

UC_Submit ..> UC_Login : <<include>>
UC_Decide ..> UC_Login : <<include>>

@enduml
```
