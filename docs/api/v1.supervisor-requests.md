## SupervisorRequests (v1)

Поток выбора научного руководителя.

Базовый путь: `/api/v1/supervisor-requests`

## Модель

`SupervisorRequestDto` (сокращённо):

```json
{
  "id": "00000000-0000-0000-0000-000000000000",
  "studentId": "00000000-0000-0000-0000-000000000000",
  "teacherUserId": "00000000-0000-0000-0000-000000000000",
  "status": {
    "id": "00000000-0000-0000-0000-000000000000",
    "codeName": "Pending",
    "displayName": "Ожидает"
  },
  "comment": "Комментарий",
  "createdAt": "2026-04-14T10:00:00Z",
  "updatedAt": null
}
```

## Эндпоинты

### `GET /api/v1/supervisor-requests`

Список запросов в контексте роли:
- `Student` — свои;
- `Teacher` — входящие;
- `Admin` — все.

### `GET /api/v1/supervisor-requests/{id}`

Детальная карточка запроса.

### `POST /api/v1/supervisor-requests`

Создание запроса студентом.

Body:

```json
{
  "teacherUserId": "00000000-0000-0000-0000-000000000000",
  "comment": "Прошу назначить научным руководителем"
}
```

### `PUT /api/v1/supervisor-requests/{id}/approve`

Подтверждение преподавателем.

### `PUT /api/v1/supervisor-requests/{id}/reject`

Отклонение преподавателем (комментарий обязателен).

Body:

```json
{
  "comment": "Не могу принять в текущем семестре"
}
```

### `PUT /api/v1/supervisor-requests/{id}/cancel`

Отмена студентом-владельцем.

## Бизнес-правила

- Нельзя создать второй `Pending` к тому же преподавателю для того же студента.
- При `approve` одного запроса автоматически отменяются другие `Pending`-запросы этого студента.
- Лимит активных `Pending` рассчитывается по количеству преподавателей кафедры студента (если кафедра задана).

## Уведомления в этом потоке

| Событие | Получатель | Тип уведомления |
|---|---|---|
| `POST /supervisor-requests` (успех) | `Teacher` | `SupervisorRequestCreated` |
| `PUT /supervisor-requests/{id}/approve` | `Student` | `SupervisorRequestStatusChanged` |
| `PUT /supervisor-requests/{id}/reject` | `Student` | `SupervisorRequestStatusChanged` |
| `PUT /supervisor-requests/{id}/cancel` | `Teacher` | `SupervisorRequestStatusChanged` |
