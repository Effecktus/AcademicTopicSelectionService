## Health

Базовые endpoints для smoke‑проверок и мониторинга.

### GET `/health`

- **Описание**: проверка доступности API (процесс запущен и отвечает).
- **Аутентификация**: не требуется.
- **Проверяет БД**: нет
- **Ответ 200**:

```json
{
  "status": "ok",
  "environment": "Development",
  "utc": "2026-02-03T12:34:56.789+00:00"
}
```

### GET `/health/db`

- **Описание**: проверка доступности PostgreSQL из API (`Database.CanConnectAsync`).
- **Аутентификация** (любой из вариантов):
  1. Заголовок **`Authorization: Bearer <accessToken>`** с JWT пользователя с ролью **`Admin`**,  
  **или**
  2. Заголовок **`X-Health-Probe-Key: <секрет>`**, если в конфигурации API задан непустой **`Health:DbProbeKey`** (совпадение с секретом по байтам, без логирования значения).
- Если ни условие 1, ни условие 2 не выполнены — **401 Unauthorized**.
- **Ответ 200** (при успешной авторизации и проверке):

```json
{
  "status": "ok",
  "db": "postgres",
  "canConnect": true
}
```

**Практика для Docker / оркестратора:** либо вызывать только **`GET /health`**, либо задать **`Health:DbProbeKey`** в окружении контейнера и передавать **`X-Health-Probe-Key`** в запросе к **`/health/db`**.
