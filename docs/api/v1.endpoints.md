# API v1: Актуальные endpoints

Снимок актуального набора endpoint-ов backend на `2026-05-06`.

## Health

- `GET /health`
- `GET /health/db`

## Auth

- `POST /api/v1/auth/login`
- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/logout`

## Пользователи

- `POST /api/v1/users` — создание пользователя (только Admin)

## Справочники (CRUD)

Для каждого ресурса поддерживаются `GET list`, `GET by id`, `POST`, `PUT`, `PATCH`, `DELETE`.

| Ресурс | Базовый путь |
|--------|-------------|
| Роли пользователей | `/api/v1/user-roles` |
| Статусы заявок | `/api/v1/application-statuses` |
| Статусы действий по заявкам | `/api/v1/application-action-statuses` |
| Статусы тем | `/api/v1/topic-statuses` |
| Типы создателей тем | `/api/v1/topic-creator-types` |
| Типы уведомлений | `/api/v1/notification-types` |
| Учебные группы | `/api/v1/study-groups` |
| Учёные степени | `/api/v1/academic-degrees` |
| Учёные звания | `/api/v1/academic-titles` |
| Должности | `/api/v1/positions` |

## Доменный read API

- `GET /api/v1/teachers`
- `GET /api/v1/teachers/{id}`
- `GET /api/v1/students`
- `GET /api/v1/students/{id}`

## Topics

- `GET /api/v1/topics` — фильтры: `query`, `creatorQuery`, `statusCodeName`, `createdByUserId`, `creatorTypeCodeName`, **`createdFromUtc`**, **`createdToUtc`**, `sort`, `page`, `pageSize` (подробнее: [v1.topics.md](v1.topics.md))
- `GET /api/v1/topics/{id}`
- `POST /api/v1/topics`
- `PUT /api/v1/topics/{id}`
- `PATCH /api/v1/topics/{id}`
- `DELETE /api/v1/topics/{id}`

## Поток 1: SupervisorRequests

- `GET /api/v1/supervisor-requests` — опционально: **`createdFromUtc`**, **`createdToUtc`**, `sort`, `page`, `pageSize` (см. [v1.supervisor-requests.md](v1.supervisor-requests.md))
- `GET /api/v1/supervisor-requests/{id}`
- `POST /api/v1/supervisor-requests`
- `PUT /api/v1/supervisor-requests/{id}/approve`
- `PUT /api/v1/supervisor-requests/{id}/reject`
- `PUT /api/v1/supervisor-requests/{id}/cancel`

## Поток 2: StudentApplications

- `GET /api/v1/applications`
- `GET /api/v1/applications/{id}` — деталь с фильтром видимости по роли (см. `v1.applications.md`)
- `POST /api/v1/applications`
- `PUT /api/v1/applications/{id}/approve` — научрук: одобрение и автоматическая передача заведующему (`Pending` → `PendingDepartmentHead`)
- `PUT /api/v1/applications/{id}/reject`
- `PUT /api/v1/applications/{id}/department-head-approve`
- `PUT /api/v1/applications/{id}/department-head-reject`
- `PUT /api/v1/applications/{id}/cancel`

## Notifications

- `DELETE /api/v1/application-actions/{id}`

- `GET /api/v1/applications/{applicationId}/messages`
- `POST /api/v1/applications/{applicationId}/messages`
- `PUT /api/v1/applications/{applicationId}/messages/read-all`

## GraduateWorks (Архив ВКР)

- `GET /api/v1/graduate-works`
- `GET /api/v1/graduate-works/{id}`
- `POST /api/v1/graduate-works` — создание записи (Admin)
- `PUT /api/v1/graduate-works/{id}` — обновление (Admin)
- `DELETE /api/v1/graduate-works/{id}` — удаление (Admin)
- `POST /api/v1/graduate-works/{id}/upload-url/{fileType}` — получить presigned URL для загрузки (Admin)
- `POST /api/v1/graduate-works/{id}/confirm-upload/{fileType}` — подтвердить загрузку файла (Admin)
- `GET /api/v1/graduate-works/{id}/download-url/{fileType}` — получить presigned URL для скачивания
