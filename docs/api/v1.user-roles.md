## UserRoles (v1)

Базовый CRUD для справочника ролей пользователей. Это “эталон” для остальных CRUD‑контроллеров.

Базовый путь: `/api/v1/user-roles`

### Модель

`UserRoleDto`:

```json
{
  "id": "00000000-0000-0000-0000-000000000000",
  "name": "Student",
  "displayName": "Студент",
  "createdAt": "2026-02-03T12:34:56.789Z",
  "updatedAt": null
}
```

### GET `/api/v1/user-roles`

Список (пагинация + поиск).

**Query params**:
- `searchString` (string, optional): поиск по `name` / `displayName`
- `page` (int, optional, default=1)
- `pageSize` (int, optional, default=50, 1..200)

**Ответ 200**:

```json
{
  "page": 1,
  "pageSize": 50,
  "total": 4,
  "items": []
}
```

### GET `/api/v1/user-roles/{id}`

Получить роль по `id`.

- **Ответ 200**: `UserRoleDto`
- **Ответ 404**: роль не найдена (ProblemDetails)

### POST `/api/v1/user-roles`

Создать роль.

**Body**:

```json
{
  "name": "NewRole",
  "displayName": "Новая роль"
}
```

- **Ответ 201**: `UserRoleDto`
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### PUT `/api/v1/user-roles/{id}`

Обновить роль.

**Body**:

```json
{
  "name": "NewRole",
  "displayName": "Обновлённая роль"
}
```

- **Ответ 200**: `UserRoleDto`
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 404**: роль не найдена (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### PATCH `/api/v1/user-roles/{id}`

Частично обновить роль. Передаются только поля, которые нужно изменить.

**Body**:

```json
{
  "name": null,
  "displayName": "Обновлённое отображаемое имя"
}
```

- **Ответ 200**: `UserRoleDto`
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 404**: роль не найдена (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### DELETE `/api/v1/user-roles/{id}`

Удалить роль.

- **Ответ 204**: удалено
- **Ответ 404**: роль не найдена (ProblemDetails)
