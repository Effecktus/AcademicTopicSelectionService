## Applications (v1)

Поток утверждения темы ВКР.

Базовый путь: `/api/v1/applications`

## Ключевое правило

Создание заявки возможно только с одобренным `SupervisorRequest`:
- в `POST /api/v1/applications` обязательно передаётся `supervisorRequestId`;
- научный руководитель в этом потоке определяется по `SupervisorRequest.TeacherUserId`.

## Эндпоинты

### `GET /api/v1/applications`

Список заявок (фильтрация по роли текущего пользователя).

### `GET /api/v1/applications/{id}`

Детальная заявка с историей действий. Доступ только к заявкам, которые пользователь имеет право видеть по роли (как в списке `GET /applications`): студент — свои; преподаватель — где он научрук; заведующий — кафедра научрука; админ — все. Иначе ответ **404** (без признака существования чужой записи).

### `POST /api/v1/applications`

Создание заявки студентом.

Body (один из двух вариантов):

```json
{
  "topicId": "00000000-0000-0000-0000-000000000000",
  "supervisorRequestId": "00000000-0000-0000-0000-000000000000"
}
```

```json
{
  "supervisorRequestId": "00000000-0000-0000-0000-000000000000",
  "proposedTitle": "Тема, предложенная студентом",
  "proposedDescription": "Краткое описание"
}
```

Правило валидации:
- нужно передать либо `topicId`, либо `proposedTitle`;
- передавать одновременно оба поля нельзя.

### `PUT /api/v1/applications/{id}/approve`

Научрук одобряет заявку и **сразу передаёт её заведующему кафедрой**: `Pending` → `PendingDepartmentHead` (промежуточный статус `ApprovedBySupervisor` в этом сценарии не используется).

Тело запроса — JSON, можно передать пустой объект `{}` или опционально:

```json
{ "comment": "Краткий комментарий научрука" }
```

Комментарий попадает в запись действия и в текст уведомления заведующему; пустое значение допустимо.

### `PUT /api/v1/applications/{id}/reject`

Научрук отклоняет: `Pending` → `RejectedBySupervisor` (комментарий обязателен).

### `PUT /api/v1/applications/{id}/department-head-approve`

Заведующий утверждает: `PendingDepartmentHead` → `ApprovedByDepartmentHead`.

Тело: `{}` или опционально `{ "comment": "..." }`.

### `PUT /api/v1/applications/{id}/department-head-reject`

Заведующий отклоняет: `PendingDepartmentHead` → `RejectedByDepartmentHead` (комментарий обязателен).

### `PUT /api/v1/applications/{id}/cancel`

Студент отменяет заявку:
- разрешено из `OnEditing`, `Pending` и `ApprovedBySupervisor` (последний код в справочнике сохраняется для совместимости; при текущем потоке после одобрения научрука статус сразу `PendingDepartmentHead`, отмена недоступна);
- из `PendingDepartmentHead` и финальных статусов — **409 Conflict** (недопустимый переход).

## Лимит научрука

Проверяется при `department-head-approve`:
- учитываются только заявки научрука со статусом `ApprovedByDepartmentHead`;
- если `count >= Teachers.MaxStudentsLimit`, операция отклоняется.

## Уведомления в этом потоке

| Событие | Получатель | Тип уведомления |
|---|---|---|
| `POST /applications` (успех) | `Teacher` из `SupervisorRequest` | `ApplicationSubmittedToSupervisor` |
| `PUT /applications/{id}/approve` | `DepartmentHead` кафедры научрука | `ApplicationSubmittedToDepartmentHead` |
| `PUT /applications/{id}/reject` | `Student` | `ApplicationStatusChanged` |
| `PUT /applications/{id}/department-head-approve` | `Student`, научрук (`SupervisorRequest.TeacherUserId`) | `ApplicationStatusChanged`, `SupervisorDecisionByDepartmentHead` |
| `PUT /applications/{id}/department-head-reject` | `Student`, научрук | `ApplicationStatusChanged`, `SupervisorDecisionByDepartmentHead` |
