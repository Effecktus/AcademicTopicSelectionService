## Topics (v1)

Каталог тем ВКР: создание преподавателем или студентом (тип создателя задаётся при создании), просмотр и изменение по правам автора.

Базовый путь: `/api/v1/topics`

Все методы под `[Authorize]`.

## Модель списка / деталей

`TopicDto` (сокращённо):

```json
{
  "id": "00000000-0000-0000-0000-000000000000",
  "title": "Тема ВКР",
  "description": "Описание",
  "status": { "id": "…", "codeName": "Active", "displayName": "Активна" },
  "creatorType": { "id": "…", "codeName": "Teacher", "displayName": "Преподаватель" },
  "createdByUserId": "00000000-0000-0000-0000-000000000000",
  "createdByEmail": "teacher@example.com",
  "createdByFirstName": "Иван",
  "createdByLastName": "Иванов",
  "createdAt": "2026-04-27T12:00:00Z",
  "updatedAt": null
}
```

Ответ списка: `PagedResult<TopicDto>` — поля `page`, `pageSize`, `total`, `items`.

## Эндпоинты

### `GET /api/v1/topics`

Список тем с фильтрами, сортировкой и пагинацией.

| Query-параметр | Тип | Описание |
|----------------|-----|----------|
| `query` | string? | Подстрока в **названии** темы (ILIKE, без описания) |
| `creatorQuery` | string? | Подстрока в ФИО или email автора (ILIKE) |
| `statusCodeName` | string? | Точное совпадение кода статуса, например `Active`, `Inactive` |
| `createdByUserId` | Guid? | Только темы указанного автора |
| `creatorTypeCodeName` | string? | Код типа создателя: `Teacher`, `Student` |
| `createdFromUtc` | DateTimeOffset? | Нижняя граница **даты создания** записи (UTC, включительно) |
| `createdToUtc` | DateTimeOffset? | Верхняя граница **даты создания** записи (UTC, включительно) |
| `sort` | string? | См. таблицу сортировки ниже |
| `page` | int | Номер страницы с 1 (по умолчанию 1) |
| `pageSize` | int | Размер страницы 1–200 (по умолчанию 50) |

**Сортировка (`sort`):** `createdAtDesc` (по умолчанию), `createdAtAsc`, `titleAsc`, `titleDesc`, `statusAsc`, `statusDesc`, `creatorAsc`, `creatorDesc`, `creatorTypeAsc`, `creatorTypeDesc`.

Рекомендуемый формат дат в query: ISO 8601 (`…Z` или offset), например `2026-01-01T00:00:00.000Z`.

### `GET /api/v1/topics/{id}`

Детальная карточка темы.

### `POST /api/v1/topics`

Создание темы (роль и бизнес-правила — по политике API; типично преподаватель).

### `PUT /api/v1/topics/{id}`

Полное обновление темы (все поля тела обязательны по контракту `ReplaceTopicCommand`).

### `PATCH /api/v1/topics/{id}`

Частичное обновление (`null` в поле — не менять).

### `DELETE /api/v1/topics/{id}`

Удаление темы (при отсутствии блокирующих связей — см. ответы API).

## Связь с фронтендом

- SPA передаёт в список те же query-параметры, что и в таблице выше.
- Для диапазона дат создания клиент обычно отправляет границы дня в UTC (начало 00:00:00 и конец 23:59:59.999 локального дня, преобразованные в ISO).
