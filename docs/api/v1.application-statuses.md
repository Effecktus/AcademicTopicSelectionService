## Application Statuses (v1)

CRUD для справочника статусов заявки. Структура аналогична User Roles.

Базовый путь: `/api/v1/application-statuses`

### Модель

`ApplicationStatusDto`:

```json
{
  "id": "00000000-0000-0000-0000-000000000000",
  "name": "Draft",
  "displayName": "Черновик",
  "createdAt": "2026-02-03T12:34:56.789Z",
  "updatedAt": null
}
```

### GET `/api/v1/application-statuses`

Список (пагинация + поиск).

**Query params**:
- `searchString` (string, optional): поиск по `name` / `displayName`
- `page` (int, optional, default=1)
- `pageSize` (int, optional, default=50, 1..200)

**Ответ 200**:

```json
{
  "page": 1,
  "pageSize": 50,
  "total": 0,
  "items": []
}
```

**Ответ 400**: ошибка валидации параметров (ProblemDetails).

### GET `/api/v1/application-statuses/{id}`

Получить статус заявки по `id` (GUID).

- **Ответ 200**: `ApplicationStatusDto`
- **Ответ 404**: статус не найден (ProblemDetails)

### POST `/api/v1/application-statuses`

Создать статус заявки.

**Body**:

```json
{
  "name": "NewStatus",
  "displayName": "Новый статус"
}
```

- **Ответ 201**: `ApplicationStatusDto`, заголовок `Location` — ссылка на созданный ресурс
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### PUT `/api/v1/application-statuses/{id}`

Полностью обновить статус заявки. Все поля обязательны.

**Body**: как у POST.

- **Ответ 200**: `ApplicationStatusDto`
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 404**: статус не найден (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### PATCH `/api/v1/application-statuses/{id}`

Частично обновить статус заявки. Передаются только поля, которые нужно изменить.

**Body**:

```json
{
  "name": null,
  "displayName": "Обновлённое отображаемое имя"
}
```

- **Ответ 200**: `ApplicationStatusDto`
- **Ответ 400**: ошибка валидации (ProblemDetails)
- **Ответ 404**: статус не найден (ProblemDetails)
- **Ответ 409**: конфликт по `name` (ProblemDetails)

### DELETE `/api/v1/application-statuses/{id}`

Удалить статус заявки.

- **Ответ 204**: удалено
- **Ответ 404**: статус не найден (ProblemDetails)
